## Vision

### 10x Check
The 10x version is "Zen Caregiving Volunteer Platform" -- automated scheduling based on preferences, SMS for non-tech-savvy volunteers, volunteer hour tracking for grant reporting, multi-site support, and a public-facing recruitment page. Current scope is right for v1.

## Scope Decisions

| # | Proposal | Effort | Decision | Reasoning |
|---|----------|--------|----------|-----------|
| 1 | Toast component (replace alert()) | S | ACCEPTED | alert() blocks UI, toasts are non-blocking and modern |
| 2 | Offline attendance queue | M | ACCEPTED | Volunteers at care facility may have spotty WiFi |
| 3 | Shift partner display in My Shifts | S | ACCEPTED | Zero-cost UX win, data already in cache |
| 4 | Copy schedule + calendar feed (.ics) + guide docs | S | ACCEPTED | Non-technical volunteers prefer SMS/calendar |
| 5 | Weekly digest email | S | ACCEPTED | One email vs many per-shift reminders |
| 6 | Admin notification dashboard | M | SKIPPED | Resend has its own dashboard |

## Hardening Scope (baseline)

- **Error handling wrapper** (sbQuery) for all client-side Supabase calls + toast on failure
- **Double-click protection** on all save buttons (disable during async)
- **Loading spinner** on first page load
- **Dedup away alerts** (only fire when status changes TO away, not on re-saves)
- **Email verification flow** (confirmation email before storing address)
- **Edge Function tests** (Deno tests with mocked Supabase + Resend)
- **Deploy docs** in CLAUDE.md

## Build Now (added during review)

- Attendance trends (4-week history) in admin dashboard
- Admin shift assignment management UI
- Deploy documentation in CLAUDE.md

## Architecture

```
  GitHub Pages (index.html + app.js + sw.js)
       │ anon key
       ▼
  Supabase
  ├── PostgREST (CRUD with RLS)
  ├── Auth (magic link for admin)
  ├── Edge Functions
  │   ├── update-email (+ verification flow)
  │   ├── away-alert (email preferred volunteers)
  │   ├── shift-reminder (cron-triggered)
  │   ├── weekly-digest (cron-triggered, NEW)
  │   ├── calendar-feed (.ics endpoint, NEW)
  │   └── verify-email (confirmation handler, NEW)
  ├── PostgreSQL + RLS + pg_cron
  └── Resend API (email delivery)
```

## Deployment Checklist

1. Configure Supabase secrets: `RESEND_API_KEY`, `FROM_EMAIL`, `APP_URL`
2. Configure database settings: `app.settings.supabase_url`, `app.settings.service_role_key`
3. Deploy Edge Functions: `supabase functions deploy`
4. Push DB migrations: `supabase db push`
5. Push frontend: `git push origin main` (GitHub Pages auto-deploys)
6. Verify: check cron jobs in Supabase dashboard, send test email
